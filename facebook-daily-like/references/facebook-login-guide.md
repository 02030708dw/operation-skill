# Facebook 登录与连接排查（MYT Android）

此文档只用于排查。登录必须由用户在设备画面中人工完成，skill 不应收集、保存或输入 Facebook 密码、短信验证码或双因素验证码。

## 推荐顺序

1. 确认 Hermes 所在电脑能访问 MYT 控制器。
2. 使用 skill 脚本的 `--check` 检查目标端口。
3. 在 MYT 的设备画面中打开 Facebook。
4. 若未登录，由用户本人完成登录和所有安全验证。
5. 登录成功并能看到动态后，再执行 skill 的预演。
6. 预演能识别未点赞按钮后，才执行真实点赞。

## 常见问题

### MYT API 无法连接

- 检查 `MYT_HOST` 是否只包含主机名或 IP，不包含协议和端口。
- 检查电脑与 MYT 是否处于可互通网络。
- 检查实例端口映射。默认规则是 T1001=10005、T1002=10008，每台递增 3；自定义映射时直接传端口。
- 检查 Windows 防火墙、路由器隔离或 VPN 是否阻断局域网连接。

### Facebook 停留在登录页

不要尝试通过固定坐标自动填写账号密码。Facebook 登录页面会变化，并可能要求验证码、设备确认或双因素验证。请在设备画面中人工完成。

### Facebook 可以打开但没有动态

- 检查设备网络和代理设置。
- 在设备中手动刷新，确认账号本身可以正常加载动态。
- 若 Facebook 显示安全检查或异常登录提示，先人工处理。

### 识别不到点赞按钮

- 使用 `--verbose` 查看脚本发现的可点击节点描述。
- 确认 Facebook 语言是中文简体、中文繁体或英文。
- 确认动态中确实存在未点赞的普通帖子。
- Facebook 更新文案后，应更新脚本的包含/排除标签，不应改成固定坐标。

### 预演成功但执行未达到数量

- 页面可能在点击后刷新或弹出遮罩。
- 降低 `--count`，观察单次操作。
- 增加 `--max-cycles` 前先检查详细日志，避免无意义地反复滚动。
- 在 MYT 设备画面中人工确认实际点赞状态。
